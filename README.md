## 🏚️ The Downfalls of One-Swoop Creation

Creating an entire portfolio in a single sprint teaches you exactly why software engineers invented the concept of "iterations":

1. **The Hallucination Graveyard** — During the initial refactor, we accidentally left behind three entirely fictional projects (Dispute Resolution Engine, AP Data Integrity Validator, Revenue Growth Stack Template). None of these exist. We caught them later. You might find others.

2. **Contact Link Archaeology** — We had to hunt through four HTML files to find placeholder email addresses and generic GitHub/LinkedIn URLs. In a proper design flow, this would have been solved once, at the source. Instead, we played digital whack-a-mole.

3. **The Thumbnail Principle** — Building everything at once means you're working at 10,000 feet. You see the shape of the thing, but you miss the pixel-level details. Those details bite you later.

4. **AI Agent Drift** — The Antigravity IDE's agent manager (running Claude Opus 4.6 with extended thinking) made decisions about content that now require a full forensic review and a detailed refactor prompt. We're not fixing those *right now* because the user needs to review comprehensively first.

5. **The "Just One More Thing" Trap** — Every addition requires three follow-up decisions. You commit, push, then realize you need to add meta descriptions, update footer copy, verify accessibility attributes... it never ends.

**The lesson:** Building software in real-time is like writing with a pen instead of a pencil. It works. It's fast. But you're going to want an eraser.

## 📖 What This Project Actually Is

This is a **vanilla HTML/CSS/JavaScript portfolio and engineering blog** hosted on GitHub Pages, designed to document real, open-source projects and operational problem-solving frameworks.

### The Core Idea

Instead of a traditional resume or marketing site:
- **No frameworks** (vanilla tech stack only)
- **No external dependencies** (zero vendor bloat)
- **No tracking scripts** (your data stays yours)
- **No corporate jargon** (we call things by their actual names)
- **No sales pitch** (just documentation of real work)

The site serves as an archive of:
- **Engineering logs:** Technical post-mortems and architecture breakdowns
- **Open-source projects:** 10+ real GitHub repositories with verifiable code
- **Operational frameworks:** Reusable patterns for workflow automation and AI integration

### The Vanilla Architecture

```
index.html          → Blog feed and portfolio home
projects.html       → 11 real GitHub repositories (ARM64brute2020, autonomous-edge-ingestion-pipeline, etc.)
tools.html          → Roadmap of upcoming client-side utilities
contact.html        → Direct communication layer
style.css           → Pure CSS variables, no preprocessor
app.js              → Vanilla JavaScript, no bundler
```

**Zero build steps.** No npm install. No webpack. No framework runtime. Just HTML, CSS, and JavaScript that runs directly in your browser—exactly as intended by the web standards committee in 1995... but with dark mode and semantic accessibility.
